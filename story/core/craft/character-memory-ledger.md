# Character Memory Ledger

*Keeps "who knows what" consistent when writing or revising scenes: produces per-character memory ledgers and memory deltas after each scene.*

Use this doc when you need to:
- Track what each character will remember after a scene
- Prevent contradictions in later dialogue (knowledge, debts, threats)
- Design believable misunderstandings and misreads

This is writing-only. Primary template:
- <STORY_ROOT>/templates/character_memory_ledger.md

## Hard rules

1) Separate: know vs assume vs misread.
2) Memory entries must be actionable in later scenes (a trigger, a leverage, a constraint).
3) If a character did not witness it, they cannot "know" it unless you add an information transfer.

## Output format (always)

### 1) Memory ledger (per involved character)
For each involved character, fill:
- knows (high confidence)
- assumes (medium)
- misreads (wrong but plausible)
- debts/promises/threats
- triggers
- next-scene default posture

### 2) Memory deltas
List only what changed this scene (so it can be appended to a running doc).

### 3) Consistency checks
- What line would break if we forgot this memory?
- What future reveal is blocked if we give them too much knowledge now?

## Recommended storage (repo)

- Keep a running doc per major character under <STORY_ROOT>/ (one file per character).
- When unsure, write the delta first, then fold it into the running doc.
