# Antagonist Pressure Design

*For designing pursuers/antagonist groups: produces a pressure matrix, objectives, beliefs, constraints, evidence ladder, escalation ladder, and failure mode. Use whenever an antagonist must behave rationally across scenes.*

Use this doc when you need to:
- Make an antagonist group behave rationally across scenes
- Define why they escalate (and why they sometimes do not)
- Support fight triggers with clear evidence and constraints

This is writing-only. Primary template:
- <STORY_ROOT>/templates/antagonist_pressure_matrix.md

## Hard rules

1) Give the group 1 primary objective and 1 secondary objective.
2) Add at least 1 hard constraint that prevents "just kill everyone".
3) Include an evidence ladder: what they see changes what they believe, which changes tactics.
4) Include an escalation ladder: each step has a trigger and a cost.

## Output format (always)

### 1) Pressure matrix (filled)
Fill the template fields as bullets.

### 2) Scene hooks
List 3–5 short hooks you can drop into prose to show pressure without exposition:
- procedural orders
- search behavior
- interrogation pattern
- threat style

### 3) Integration note
State how this matrix forces the fight trigger and the fight stop condition for the current scene.
