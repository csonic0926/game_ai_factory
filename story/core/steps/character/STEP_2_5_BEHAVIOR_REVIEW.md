# STEP 2.5 — Behavior Review

## Purpose

Review the saved behavior baseline and decide whether it is ready to use as a character-writing reference.

## Read inputs from

- `<STORY_ROOT>/state/character_baselines/CHARACTER_BEHAVIOR_AND_VOICE.md`
- `<STORY_ROOT>/state/character_baselines/CHARACTER_CONCEPT.md`
- `<STORY_ROOT>/state/character_baselines/CHARACTER_WORLD_ROLE.md`

## Save output to

- `<STORY_ROOT>/state/character_baselines/CHARACTER_BEHAVIOR_AND_VOICE_REVIEW.md`

## Skill use

- No skill required for this step.

## Task

Read the saved behavior baseline, compare it against the concept and world-role references, and record a local review verdict.

Judge the baseline only on what is written. Do not invent missing behavior. If the baseline is weak, say exactly what is missing and why that prevents use in scene writing.

## Required output blocks

Write the review artifact with these blocks in this order:

1. `## Verdict`
2. `## Review Notes`
3. `## Blockers` only when the verdict is `FAIL`

### Verdict block

The verdict block must contain exactly one of:

- `STEP 2.5 PASS`
- `STEP 2.5 FAIL`

If the result is `FAIL`, include one short blocker sentence immediately under the verdict.

### Review Notes block

Use short bullet points that state what was checked and what was found.
Keep the notes concrete and tied to the source text.

### Blockers block

Include this block only when the review fails.

- List the specific reason or reasons the baseline cannot pass.
- Each blocker must be actionable and concrete.
- Each blocker must point to the missing or unusable behavior detail, not just repeat a label like “too vague.”

## Review checks

Mark the step as passing only if all checks below are satisfied.

### Observable habits

- The character can be identified through visible actions, routines, or habits.
- The habits are specific enough to stage in a scene without extra invention.

### Pressure reaction

- The character has a clear reaction pattern under pressure, conflict, or uncertainty.
- The reaction pattern can be used directly when writing future scenes.

### Speaking pattern

- The speech pattern is distinct enough to guide later dialogue.
- The speech pattern is described through concrete behavior, phrasing, pace, or manner, not only abstract personality labels.

### Fit with world position

- The behavior matches the character's concept and world role.
- The behavior feels like it belongs to the character's daily life and social position.

### Avoidances

- The baseline includes readable do-not behaviors, boundaries, or refusals.
- The avoidances help define scene behavior instead of only listing abstract taboos.

## Pass / fail rules

- Pass only when every review check above is satisfied.
- Fail when any check cannot be verified from the source text.
- Fail when the baseline would still leave later writers guessing how the character behaves in scene.

## Stop condition

- End with the required verdict line.
- If `FAIL`, include the blocker(s) before ending the review.
