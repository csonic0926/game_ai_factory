# STEP 0 — Character Concept Framing

## Purpose

Produce one original character concept baseline that later character-creation steps can extend without re-framing.

## Read inputs from

- the user's character brief for this character request
- any inspiration anchor named by the user
- packaged characters under `<STORY_ROOT>/state/characters/`, if any exist
- cast-management artifacts when they exist and constrain this request

## Save output to

- `<STORY_ROOT>/state/character_baselines/CHARACTER_CONCEPT.md`

## Skill use

- No skill required for this step.

## Task

Read the brief, compare it against any existing packaged characters, and write a single character concept that can support later world-position, behavior, knowledge, and relations work.

The concept must make clear:

- what function this character serves in the story world
- how other people would ordinarily read this person
- what daily-life pressure shapes the character most
- what visible contradiction makes the character story-usable
- how this character differs in story function from already-packaged characters, if any exist

When the file is complete:

- the concept must be original, specific, and usable by later steps without extra interpretation
- the concept must stay at this framing level; do not add backstory, scene beats, or final biography details
- if packaged characters exist, the new concept must be clearly distinguishable from the closest one in story function

## Required output blocks

### `CHARACTER PREMISE`

Write one concise paragraph stating the character's core idea, story function, and why this concept matters.

### `SOCIAL READING`

Write one concise paragraph describing how ordinary people in the world would usually read, classify, or misread this person.

### `DAILY-LIFE PRESSURE`

Write one concise paragraph naming the main ordinary pressure, obligation, or difficulty that shapes this character's life.

### `VISIBLE CONTRADICTION`

Write one concise paragraph naming one contradiction that is visible in the character's role, behavior, position, or daily life, and why that contradiction is useful for story.

### `DISTINCTNESS NOTE`

If packaged characters already exist, name the closest existing character and state exactly how this concept differs in story function.

If no packaged characters exist, write `None`.
