# STEP 3.5 — Rebalance Acceptance

## Purpose

Review the saved rebalance pass and decide whether it is ready to support cast-action requests.

## Read inputs from

- `<STORY_ROOT>/state/cast_management/CAST_REBALANCE.md`
- `<STORY_ROOT>/state/cast_management/CAST_MISSING_AND_OVERLAP.md`
- `<STORY_ROOT>/state/cast_management/CAST_AUDIT.md`

## Save output to

- `<STORY_ROOT>/state/cast_management/CAST_REBALANCE_REVIEW.md`

## Skill use

- No skill required for this step.

## Task

Check whether the saved rebalance pass names the important relation and pressure problems clearly enough for the next cast step to act on them.

## Acceptance criteria

### Relation gaps

This step passes when:

- each important relation gap is named in concrete cast terms
- the rebalance pass shows which character links are one-sided, missing, weak, or underdeveloped
- the rebalance pass makes it clear which relation problems matter most for the current cast scope

### Pressure gaps

This step passes when:

- each important pressure gap is named in concrete cast terms
- the rebalance pass shows where pressure is missing, over-clustered, or unevenly distributed
- the rebalance pass makes it clear which pressure problems matter most for the current cast scope

### Rebalance notes

This step passes when:

- the rebalance notes name the concrete change needed for each important gap
- the rebalance notes are specific enough to turn directly into a revision request or a new character request
- the rebalance notes avoid vague summary language and state the next cast move plainly

## Required stop condition

- write a short note that says `STEP 3.5 PASS` or `STEP 3.5 FAIL`
- on `FAIL`, state the blocker clearly
