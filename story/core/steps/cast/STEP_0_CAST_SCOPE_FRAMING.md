# STEP 0 — Cast Scope Framing

## Purpose

Define the current cast-management scope in a form that later cast-review steps can use directly.

## Read inputs from

- the user's cast brief
- packaged characters under `<STORY_ROOT>/state/characters/`, when they exist
- cast-management artifacts under `<STORY_ROOT>/state/cast_management/`, when they exist

## Save output to

- `<STORY_ROOT>/state/cast_management/CAST_SCOPE.md`

## Skill use

- No skill required for this step.

## Task

Turn the current cast brief and available character state into one concrete cast-review scope.

Write the scope so later steps can read the answer without guessing:

- what story support this cast pass is for
- what ensemble scale is currently needed
- whether the scope is for chapter support, arc support, or baseline world-stage support
- what story functions the cast must cover as a set
- which packaged characters are already relevant to this scope, when they exist

## Required output blocks

### `CAST PURPOSE`

State the story goal this cast pass is meant to support.

### `CURRENT SCALE TARGET`

State the current ensemble size or scale target in practical terms.

### `SCOPE LEVEL`

State whether the cast is being reviewed for chapter support, arc support, or baseline world-stage support.

### `REQUIRED STORY FUNCTIONS`

State the functions the cast must cover as a set.
