# STEP 1 — Audit Current Cast

## Purpose

Build a readable audit of the currently packaged cast for later missing-role, overlap, and rebalance steps.

## Read inputs from

- `<STORY_ROOT>/state/cast_management/CAST_SCOPE.md`
- packaged characters under `<STORY_ROOT>/state/characters/`, when they exist
- `<STORY_ROOT>/state/cast_management/CAST_CHARACTER_LIST.md`, when it exists
- `<STORY_ROOT>/state/cast_management/CHARACTER_CREATION_PROGRESS.md`, when it exists

## Save output to

- `<STORY_ROOT>/state/cast_management/CAST_AUDIT.md`

## Skill use

- No skill required for this step.

## Task

Review the current packaged characters and turn them into one cast audit.

Make the audit usable on its own so later steps can read:

- who currently exists in the cast
- what each character functionally covers
- what pressure structure each character contributes
- which important relation links already exist
- where obvious gaps or overlaps already appear

For each character that matters to this cast pass, give a short functional summary that names the character, the role they cover, the pressure they carry, and any relation they establish.

## Required output blocks

### `CURRENT CAST`

List the packaged characters that matter to this cast pass.

### `ROLE COVERAGE`

State the story functions already covered by the current cast.

### `KNOWN GAPS`

State the important cast functions, relations, or pressures that are missing or under-covered.

### `KNOWN OVERLAPS`

State the places where two or more current characters are too similar in function, pressure, or relation shape.
