## Task

Check whether the saved missing-and-overlap diagnosis names the current cast gaps clearly enough to drive rebalance or cast-action work.

## Acceptance criteria

### Missing roles

This step passes when:

- each missing role, function, position, or pressure type is named directly
- each missing item is tied to a concrete cast need, not a general concern

### Overlap

This step passes when:

- each overlap is named directly
- each overlap explains what kind of redundancy it creates in role, pressure, or voice

### Priority order

This step passes when:

- the priority order identifies the first cast move to make
- the priority order makes clear whether the next move is create, revise, or rebalance
- the priority order gives later steps one concrete next action instead of a general problem list

- write one short note: `STEP 2.5 PASS` or `STEP 2.5 FAIL`
- on `FAIL`, name the first blocker in one sentence
