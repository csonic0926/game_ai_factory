# STEP 1 — World Rules and Constraints

## Purpose

Define the world's operating rules in a form that later world-creation steps can use directly.

## Read inputs from

- `<STORY_ROOT>/state/world_baselines/WORLD_CONCEPT.md`

## Save output to

- `<STORY_ROOT>/state/world_baselines/WORLD_RULES.md`

## Skill use

- No skill required for this step.

## Task

Turn the saved world concept into a concrete rule set that explains how action, consequence, limitation, and exception work in this world.

Write rules that later steps can use to predict:

- what people can normally do
- what systems or institutions can normally enforce
- what limits people and systems run into
- what kinds of exceptions can exist without breaking the world

## Required output blocks

### `WORLD RULES`

State the core governing rules in short, direct lines.

Each line should state one rule the world actually follows.

### `PRACTICAL CONSTRAINTS`

State the limits that ordinary people, groups, or institutions run into.

Each line should state:

- the limit
- who it applies to
- the practical effect it causes

### `CONSEQUENCE LOGIC`

State how the world usually reacts when people push against its rules or constraints.

Each line should connect an action pattern to its likely consequence.

### `KNOWN EXCEPTIONS`

List only exceptions that already belong to the world's logic.

Each exception should state:

- what rule it bends or bypasses
- why the exception can exist in this world
- what keeps the exception from swallowing the rule system

If there are no exceptions, write `None`.
