# STEP 2.5 — Geography Review

## Purpose

Check whether the saved geography is concrete enough to support later institution, movement, and twin-packaging work.

## Read inputs from

- `<STORY_ROOT>/state/world_baselines/WORLD_GEOGRAPHY.md`
- `<STORY_ROOT>/state/world_baselines/WORLD_RULES.md`

## Save output to

- `<STORY_ROOT>/state/world_baselines/WORLD_GEOGRAPHY_REVIEW.md`

## Skill use

- No skill required for this step.

## Task

Read the saved geography and decide whether it passes this review.

Write one short review result that states the decision and the blocker, if any.

## Acceptance criteria

### Geography overview

Pass when the geography states the world's major spatial layers clearly and gives a usable sense of spatial structure instead of only naming places.

### Settlement structure

Pass when inhabited zones or settlement structures are explicit, the settlement layout supports ordinary life, work, and gathering, and later steps can place institutions or routines without guessing where people are.

### Location groups

Pass when the major location groups are concrete, each location group has a readable daily-life role, and nearby relationships are clear enough to support scene staging and twin queries.

### Movement paths

Pass when ordinary movement between location groups is plausible, access paths, thresholds, or bottlenecks are concrete enough to imagine in use, and the geography supports daily routines rather than only map labels.

## Required output blocks

### `REVIEW RESULT`

Write one short note in this shape:

- `STEP 2.5 PASS` when the geography meets all criteria
- `STEP 2.5 FAIL: <blocker>` when one or more criteria are not met

Use the blocker to name the concrete gap that prevents the geography from passing.
